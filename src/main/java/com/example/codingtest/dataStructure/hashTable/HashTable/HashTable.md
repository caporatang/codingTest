# HashTable
hashTable은 key, value를 매핑할 수 있는 데이터 구조를 갖고, hash 함수를 통해, 배열에 Key에 대한 데이터를 저장할 수 있는 주소(인덱스 번호)를 계산한다.  
Key를 통해 바로 데이터가 저장되어 있는 주소를 알 수 있으므로 탐색이 빨라지며, 미리 hash 함수가 생성할 수 있는 주소(인덱스 번호)에 대한 공간을 배열로 할당한 후, Key에 따른 데이터 저장 및 탐색을 지원한다.  

## 간단한 용어 정리
1. 해쉬 함수 (Hash Function) : 데이터를 고정된 길이의 값으로 리턴해주는 함수   
    1. Hash, Hash Value, Hash Address : hash 함수를 통해 리턴된 고정된 길이의 값
2. 해쉬 테이블 (Hash Table) : 키 값의 연산에 의해 직접 접근이 가능한 데이터 구조  
   1. Slot : 해쉬 테이블에서 한 개의 데이터를 저장할 수 있는 공간

## 장단점과 활용 용도
- 장점 : 데이터 저장, 읽기 속도가 빠르고 해쉬는 Key에 대한 데이터 중복 확인이 용이하다.
- 단점 : 저장공간이 더 필요하고, 여러 키에 해당하는 주소가 동일할 경우 충돌을 해결하기 위한 별도의 자료구조가 필요하다.
- 주요 용도 :  
   - 검색이 많이 필요한 경우
   - 저장 삭제 읽기가 빈번한 경우
   - 캐쉬 구현 (중복 확인이 쉽기 때문에..)
